replicated: true   → row visible on replica (WAL streaming OK)
replicated: false  → replica lag — retry in ~1s (normal under load)

writePool.is_replica  → should be false (primary)

// CACHE-ASIDE PATTERN (Redis :6379)

          
GET  /employees  → check Redis → on miss, query read pool → store in Redis (TTL 30s)
POST /employees  → write pool → invalidate Redis key

Why? Reads hit memory instead of Postgres. Writes bust the cache so stale data expires.
